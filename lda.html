Linear Discriminant Analysis (LDA) Description: Linear Discriminant Analysis (LDA) is a supervised machine learning technique primarily used for classification and dimensionality reduction. LDA aims to find the linear combination of features that best separates two or more classes of data. Unlike PCA (Principal Component Analysis), which is unsupervised and focuses on maximizing variance, LDA is supervised and focuses on maximizing the separation between different classes in the dataset.

LDA works by finding a projection that maximizes the distance between the class means while minimizing the variance within each class. This is particularly useful for classification tasks where you want to project high-dimensional data into lower dimensions that preserve class separability.

LDA aims to maximize class separability, making it ideal for classification tasks. It projects data onto a lower-dimensional space while ensuring that classes are well-separated. Dimensionality Reduction:

LDA can reduce the number of dimensions (features) while retaining the discriminative power of the data, similar to PCA but with a focus on class separation. Computational Efficiency:

LDA is relatively computationally efficient when compared to other classification algorithms, especially in situations with fewer features and class labels. Works Well for Small Datasets:

LDA performs well on smaller datasets, especially when the classes are linearly separable. Easy to Interpret:

The results of LDA are easy to interpret. The linear combinations of features represent directions in which classes are most separable. Disadvantages of LDA: Assumes Linearity:

LDA assumes that the relationship between the features and the target variable is linear. If the relationship is non-linear, LDA might not perform well. Assumes Normality:

LDA assumes that the features within each class are normally distributed. If this assumption is violated, LDA might not give optimal results. Assumes Equal Covariance:

LDA assumes that all classes have the same covariance matrix. If the classes have significantly different covariances, the results may be suboptimal. Sensitive to Outliers:

LDA is sensitive to outliers. Extreme values in the dataset can distort the class boundaries and affect the overall performance of the model. Limitations with More Than Two Classes:

When the number of classes increases, LDA's performance may degrade, especially if the classes overlap significantly. May Not Be Suitable for Large Datasets:

LDA's assumptions about normality and linearity may not hold in large, complex datasets, where more advanced methods like Kernel Discriminant Analysis (KDA) may be more effective. When to Use LDA: Classification Tasks: LDA is ideal for classification problems, especially when you have multiple classes and the goal is to find the best features that separate them. Dimensionality Reduction: LDA can be used to reduce the number of dimensions of the data while retaining class separability for visualization or further classification tasks. Linearly Separable Data: LDA works best when the data is linearly separable, or close to being linearly separable.

# Step 1: Importing libraries
import pandas as pd
import numpy as np
import matplotlib.pyplot as plt
import seaborn as sns
from sklearn.discriminant_analysis import LinearDiscriminantAnalysis as LDA
from sklearn.preprocessing import StandardScaler
from sklearn.model_selection import train_test_split
from sklearn.linear_model import LogisticRegression
from sklearn.metrics import  classification_report

# Step 2: Load the Wine dataset
df = pd.read_csv('Wine.csv')

# Step 3: Exploratory Data Analysis (EDA)
print(df.head())

# Statistical summary of numerical features
print(df.describe())

# Visualizing numerical features with histograms
df.hist(bins=20, figsize=(10, 8))
plt.show()


sns.heatmap(df.corr(), annot=True)
plt.title('Correlation Heatmap')
plt.show()

# Step 4: Feature encoding and scaling
# Assuming all features are numerical, we will scale them.

# Separate features and target (replace 'Customer_Segment' with the actual column name)
X = df.drop('Customer_Segment', axis=1)  # Using 'Customer_Segment' as the target column
y = df['Customer_Segment']               # Target column


scaler = StandardScaler()
X_scaled = scaler.fit_transform(X)

# Step 5: Apply LDA (Linear Discriminant Analysis) to reduce dimensions to 2 components
lda = LDA(n_components=2)
X_lda = lda.fit_transform(X_scaled, y)

plt.scatter(X_lda[:, 0], X_lda[:, 1], c=y)
plt.title('LDA - 2 Components')
plt.xlabel('Linear Discriminant 1')
plt.ylabel('Linear Discriminant 2')
plt.show()

# Step 6: Train-test split
X_train, X_test, y_train, y_test = train_test_split(X_lda, y, test_size=0.3, random_state=42)

# Step 7: Model training using Logistic Regression
model = LogisticRegression()
model.fit(X_train, y_train)

# Step 8: Model prediction
y_pred = model.predict(X_test)

# Step 9: Performance metrics
print('Classification Report:\n', classification_report(y_test, y_pred))
