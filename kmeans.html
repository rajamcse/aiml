K-Means Clustering: Detailed Description, Formulae, Advantages, and Disadvantages Description of K-Means Clustering: K-Means is an unsupervised learning algorithm used for clustering. It groups data points into k clusters, where each data point belongs to the cluster whose mean is the nearest. The algorithm works iteratively to assign each data point to a cluster and then update the centroids based on the mean of the points in that cluster.

Select k initial centroids randomly or using methods like K-Means++. Assignment Step:

Each data point is assigned to the nearest centroid. The distance metric used is typically Euclidean distance. Formula for Euclidean Distance:

K-Means is computationally efficient and works well with large datasets. Simplicity:

The algorithm is easy to understand and implement. Scalability:

K-Means can handle large datasets with multiple features, especially if combined with dimensionality reduction techniques like PCA. Convergence:

K-Means converges quickly when the data is well-behaved (e.g., well-separated clusters). Interpretability:

The algorithm provides clear output with the centroids representing the "average" data points in each cluster, making it easy to interpret the results. Disadvantages of K-Means Clustering: Choosing the Right k: The user needs to specify the number of clusters (k) beforehand, which can be difficult without prior knowledge. Methods like the Elbow Method can help, but they are not always conclusive. Sensitivity to Initial Centroids: K-Means can converge to a local minimum depending on the initial choice of centroids. If the initial centroids are poorly chosen, the algorithm might result in suboptimal clustering. Assumes Spherical Clusters: K-Means assumes clusters are spherical and equally sized, which may not hold for all datasets. This can lead to poor performance when clusters have non-spherical shapes or varying sizes. Outliers: The algorithm is sensitive to outliers. Outliers can distort the centroids and affect the quality of the clustering. Non-Deterministic: Since the initial centroids are chosen randomly, the results may vary with different runs of the algorithm. When to Use K-Means Clustering: When the number of clusters is known or can be determined using methods like the Elbow Method. When the data has continuous features and is relatively clean. When you need fast and efficient clustering for large datasets.

import pandas as pd
import numpy as np
import matplotlib.pyplot as plt
import seaborn as sns
from sklearn.cluster import KMeans
from sklearn.preprocessing import StandardScaler
from sklearn.metrics import silhouette_score

df = pd.read_csv('Mall_Customers.csv')

print(df.head())

plt.subplot(1, 2, 1)
sns.histplot(df['Annual Income (k$)'])
plt.title('Annual Income Distribution')

plt.subplot(1, 2, 2)
sns.histplot(df['Spending Score (1-100)'])
plt.title('Spending Score Distribution')

plt.show()

sns.countplot(x='Gender', data=df)
plt.title('Gender Distribution')
plt.show()

# Step 3: Feature Encoding (if necessary)

# Encoding 'Gender' column from categorical to numerical values
df['Gender'] = df['Gender'].map({'Male': 1, 'Female': 0})

scaler = StandardScaler()
df[['Annual Income (k$)', 'Spending Score (1-100)']] = scaler.fit_transform(df[['Annual Income (k$)', 'Spending Score (1-100)']])


# From the Elbow plot, we choose k=5 (assumed from plot)
kmeans = KMeans(n_clusters=5)
df['Cluster'] = kmeans.fit_predict(df[['Annual Income (k$)', 'Spending Score (1-100)']])

# Step 6: Performance Metrics - Silhouette Score

# Calculating silhouette score
sil_score = silhouette_score(df[['Annual Income (k$)', 'Spending Score (1-100)']], df['Cluster'])
print(f'Silhouette Score: {sil_score}')

# Step 7: Visualizing the Clusters

# Plotting the clusters using a scatter plot
plt.figure(figsize=(10, 6))
sns.scatterplot(x='Annual Income (k$)', y='Spending Score (1-100)', hue='Cluster', data=df)
plt.title('Customer Segments')
plt.xlabel('Annual Income (k$)')
plt.ylabel('Spending Score (1-100)')
plt.show()
