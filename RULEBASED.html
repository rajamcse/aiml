Rule-Based System for Disease Identification

Description

Rule-Based Systems (RBS) are a fundamental AI approach used to model decision-making processes. These systems rely on predefined rules to process input data and produce output. During the COVID-19 pandemic, such systems were used to assess risks, allocate resources, and recommend appropriate actions based on symptoms, exposure history, and comorbidities.

Advantages

Interpretability: Easy for experts to understand and validate. Flexibility: New rules can be added as needed. Quick Decision-Making: Efficiently handles predefined scenarios.

Disadvantages

Scalability Issues: Managing many rules becomes complex. Limited Learning: Cannot adapt or improve without explicit updates. Ambiguity Handling: Struggles with conflicting or incomplete data.

# Rule-Based System for Disease Identification
def classify_risk(fever, cough, exposure, comorbidities): 
    if fever == 1 and cough == 1 and exposure == 1:
        return "High"  
    elif cough == 1 and exposure == 0:
        return "Moderate"
    elif fever == 0 and cough == 0 and exposure == 0 and comorbidities == 0:
        return "Low"
    else:
        return "Moderate"

def classify_risk(symptoms, exposure, comorbidities): 
    rules = [
        {"conditions": [1, 1, 1], "outcome": "High"},       
        {"conditions": [0, 1, 0], "outcome": "Moderate"},   
        {"conditions": [0, 0, 0], "outcome": "Low"}         
    ]
 
    for rule in rules:
        if [symptoms, exposure, comorbidities] == rule["conditions"]:
            return rule["outcome"]
    
    return "Moderate"
